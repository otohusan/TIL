## &&の注意点

### 基本

```js
function Item({ name, isPacked }) {
  return (
    <li className="item">
      {name} {isPacked && '✅'}
    </li>
  );
}
```

この場合 isPacked が true の時、チェックが追加される

### 誤った使い方

```js
function Item({ name, isPacked }) {
  return (
    <li className="item">
      {name} {0 && '✅'}
    </li>
  );
}
```

これだと、「アイテム名 0」って返ってくる（左の辺が 0 の場合は、式全体がその 0 という値に評価されてしまう？　ちょっと理解甘い）

なんにせよ、&& の左辺に数値を置くのは良くないらしい
