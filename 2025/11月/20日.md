## disabled と readonly

### disabled

- コントロールが無効化される
- フォームを Submit しても値の送信が行われない

### readonly

- 値の変更を行うことができない
- フォーカスは当てることは可能、 Submit も可能

### 余談

readonly はテキスト入力の場合のみ有効らしい、ラジオボタンに readonly を付けても値の変更ができてしまう

## 条件によって props を渡さない

例としてはこんな感じ

```ts
<button
  type="button"
  // NOTE: onAdd が渡されていない場合はクリックイベントを設定しない
  {...(onAdd ? { onClick: onAdd } : {})}
></button>
```

### 解説

```ts
onAdd ? { onClick: onAdd } : {};
```

条件に基づいて props を格納するオブジェクトを作成している

この場合だと、onAdd が存在すれば onClick をキーに onAdd を値に持つオブジェクトを作成する

```ts
{... 作成されたオブジェクト}
```

作成されたオブジェクトをスプレッド構文で展開して、コンポーネントに渡す。

onAdd が空の場合は、空のオブジェクトが渡されるから、コンポーネントには何も渡らない

### 余談

今回だと、onClick には undefined を渡すことができないので、以下のような書き方が出来なかった。

```ts
onClick={onAdd ? onAdd : undefined}
```
