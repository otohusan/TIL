### Rail index_with

- 配列から「配列の要素をキー、ブロックの戻り値をバリュー」とするハッシュを簡単に作成できる
- `to_h` でも同様のことはできるが、`index_with` の方がより意図が明確で簡潔に書ける

```rb
users = ["alice", "bob", "charlie"]

# to_h を使う場合
users.map { |name| [name, name.length] }.to_h
# => {"alice"=>5, "bob"=>3, "charlie"=>7}

# index_with を使う場合（Rails 拡張）
users.index_with { |name| name.length }
# => {"alice"=>5, "bob"=>3, "charlie"=>7}

# 固定値を入れたい場合はブロックなしでも書ける
users.index_with(0)
# => {"alice"=>0, "bob"=>0, "charlie"=>0}
```

### Ruby 疎結合に書くには

#### 依存がある状態とは

あるクラスが、他クラスの以下を知っている時

- クラス名
- クラスにあるメソッドの名前
- メソッドが要求する引数
- その引数の順番

#### 依存オブジェクトの使用

- 依存は外から渡す
- 抽象的な存在に依存する
  - インターフェースなんかのこと
  - ダックタイピングを行う際に暗示的にインターフェースを定義している

```rb
# --- Before ---
class User
  def send(user)
    # Email を直接生成して呼び出している（密結合）
    # User が Email の具体クラスと send という実装詳細を知っている
    email = Email.new
    email.send("welcome: #{user.email}")
  end
end

# --- After ---
class User
  def initialize(notifier:)
    # 初期化時に依存オブジェクトを受け取る
    @notifier = notifier
  end

  def send(user)
    message = "welcome: #{user.email}"
    # Before と異なり、send を呼べる抽象的な依存先にだけ依存する
    @notifier.send(message)
  end
end
```

#### 引数の順番に依存しないように

- キーワード引数で順番依存をなくす

```rb
# --- Before ---
def register(name, email, admin = false, send_mail = true)
  # ...
end

# 3, 4 番目の true/false が何を意味するか分かりづらい
register("taro", "taro@example.com", true, false)

# --- After ---
def register(name:, email:, admin: false, send_mail: true)
  # ...
end

register(email: "taro@example.com", name: "Taro", admin: true)
```

### Rails Lazy
- `Enumerator::Lazy` を使うと、大量のデータに対して必要な分だけ遅延評価で処理できる
- 通常の `map` や `select` は全要素を即座に評価するが、`lazy` を挟むと実際に値が必要になるまで計算しない
- 無限リストや巨大なコレクションを扱う時にメモリ効率が良い

#### 通常の処理（即時評価）
```ruby
# 全要素を先に map してから、先頭5件を取る（無駄が多い）
(1..Float::INFINITY).select { |n| n.odd? }.map { |n| n ** 2 }.first(5)
# => これは終わらない（無限に select し続ける）
```

#### lazy を使った処理（遅延評価）
```ruby
(1..Float::INFINITY).lazy.select { |n| n.odd? }.map { |n| n ** 2 }.first(5)
# => [1, 9, 25, 49, 81]
# 必要な5件分だけ評価して止まる
```

#### Rails での活用例
```ruby
# 大量のレコードを find_each で取得しつつ lazy で変換
User.find_each.lazy.select { |u| u.active? }.map { |u| u.email }.first(10)
# => アクティブなユーザーのメールアドレスを10件だけ取得
# 全レコードをメモリに載せずに済む
```

#### force で即時評価に戻す
```ruby
result = [1, 2, 3, 4, 5].lazy.select(&:odd?).map { |n| n * 10 }
# => #<Enumerator::Lazy: ...>

result.force
# => [10, 30, 50]
# force（= to_a）で遅延評価を確定させる
```
